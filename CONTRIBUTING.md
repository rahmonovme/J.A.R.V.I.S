# Contributing to J.A.R.V.I.S

Thank you for your interest in contributing to J.A.R.V.I.S! As the creator, I (@rahmonovme) welcome input, bug reports, and pull requests from from the community.

## Submitting Pull Requests (PRs)

1. **Fork the Repository**: Create your own fork of the master branch.
2. **Create a Feature Branch**: Work on a distinct, descriptive branch name (e.g., `feature/mac-audio-fix`).
3. **Write Clean Code**: Follow standard Python conventions (PEP 8).
4. **Tested Changes**: Ensure you have run J.A.R.V.I.S locally with your changes to verify they do not crash the app logic or the UI.
5. **Detailed Description**: Use the provided PR template. Explain exactly what your change does.

*Note: All PRs require a mandatory code review by the repository owner before they can be merged.*
